Porffor includes an interactive REPL (Read-Eval-Print Loop) for quick JavaScript experimentation and testing.

## Usage

Run `porf` without any arguments:

```bash
porf
```

## Examples

<CodeGroup>
```bash Start REPL
$ porf
Welcome to Porffor v0.61.10
> 
```

```bash Basic expressions
> 2 + 2
4
> Math.sqrt(16)
4
> 'hello'.toUpperCase()
'HELLO'
```

```bash Variables
> let x = 10
undefined
> x * 2
20
> const name = 'Porffor'
undefined
> name
'Porffor'
```
</CodeGroup>

## Features

The REPL provides:

<AccordionGroup>
  <Accordion title="Interactive evaluation">
    Type JavaScript expressions and see results immediately
    
    ```javascript
    > 100 / 4
    25
    > [1, 2, 3].map(x => x * 2)
    [2, 4, 6]
    ```
  </Accordion>
  
  <Accordion title="Multi-line input">
    Write functions and complex expressions across multiple lines
    
    ```javascript
    > function greet(name) {
    ...   return `Hello, ${name}!`;
    ... }
    undefined
    > greet('World')
    'Hello, World!'
    ```
  </Accordion>
  
  <Accordion title="Variable persistence">
    Variables remain available throughout the session
    
    ```javascript
    > let counter = 0
    undefined
    > counter++
    0
    > counter++
    1
    > counter
    2
    ```
  </Accordion>
  
  <Accordion title="Built-in commands">
    Special REPL commands for convenience
    
    ```javascript
    > .help      // Show help
    > .clear     // Clear context
    > .exit      // Exit REPL
    ```
  </Accordion>
</AccordionGroup>

## REPL Commands

<ParamField path=".help" type="command">
  Show available REPL commands and help information
  
  ```javascript
  > .help
  ```
</ParamField>

<ParamField path=".clear" type="command">
  Clear the REPL context (reset all variables and functions)
  
  ```javascript
  > let x = 10
  > x
  10
  > .clear
  Clearing context...
  > x
  ReferenceError: x is not defined
  ```
</ParamField>

<ParamField path=".exit" type="command">
  Exit the REPL (same as Ctrl+D or Ctrl+C twice)
  
  ```javascript
  > .exit
  ```
</ParamField>

<ParamField path=".break" type="command">
  Cancel current multi-line input
  
  ```javascript
  > function incomplete() {
  ... .break
  >
  ```
</ParamField>

## Keyboard Shortcuts

<ParamField path="Ctrl+C" type="shortcut">
  - Once: Cancel current line
  - Twice: Exit REPL
</ParamField>

<ParamField path="Ctrl+D" type="shortcut">
  Exit REPL (on empty line)
</ParamField>

<ParamField path="Up/Down Arrows" type="shortcut">
  Navigate command history
</ParamField>

<ParamField path="Tab" type="shortcut">
  Auto-completion (if supported)
</ParamField>

## Implementation

Porffor's REPL uses Node.js's built-in `node:repl` module, providing:

- Standard REPL functionality
- Command history
- Tab completion (limited)
- Multi-line editing
- Syntax highlighting (terminal dependent)

## Limitations

<Warning>
Porffor's REPL has the same limitations as regular Porffor execution:
</Warning>

- Limited async support (Promise/await bugs)
- No variables between scopes (except args and globals)
- No `eval()` or `Function()` (AOT compilation)
- Limited standard library support

## Troubleshooting

### Unexpected behavior

```javascript
// Clear context and try again
> .clear
Clearing context...
```

### Exit not working

```bash
# Press Ctrl+C twice
# Or press Ctrl+D on empty line
# Or type .exit
```

### Multi-line input stuck

```javascript
// Use .break to cancel
> function incomplete() {
... .break
>
```
